Outlined below are a few explainations of my process, technological stack, and development style.


Conception

I like sketching out ideas on paper before actually putting in any work. I'll usually just do a few really bad sketches with pen and markers and go from there.

Design

I either design in Photoshop, Illustrator, or (more typically) straight in the editor depending on the scenario and style of the project.

CLI

I use gulp for compilation of my SASS, minification/validation of my js, polyfills for ES6, and inlining my svg sprites. I use browerSync to set up live reloading of my page during developlment.

Styles

I write my CSS in SASS (SCSS) and compile it using libSASS and autoprefixer. If the project won't be able to support flexbox, I use Susy for SASS to manage my grid. I am in the process of exploring the use of javascript for styles in place of CSS.

Interactivity

If the project is complex enough to need a javascript framework, I generally lean towards AngularJS, but I'm looking for a reason to learn React and Ember. I have no loyalty to a specific framework, but instead just want to use the best tool for each individual job.

Backend

I currently use Laravel + MySQL for my backend on projects, but I am hoping to try a node.js + mongodb stack on my next big project.
